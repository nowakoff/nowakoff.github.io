Arrays vs. Hashes

When you want to store information you can use arrays or hashes. Using arrays has the most sense when you want to put your elements in certain order. Using hashes makes the most sense when you want to access elements using certain, unique label called key. Array Array indexing starts with 0. A negative index is assumed relative to the end of the array. That is, an index of -1 indicates the last element of the array, -2 is the next element and so on. animals = [‘cat’, ‘frog’, ‘dog’] Array Indices >> animals = [0] => ‘cat' You can create empty arrays by new class method: months =Array.new or months = [] Hash A hash is a collection of key-value pairs like this “pet” => “cat”. It is similar to array, expect that indexing is made by keys name, not integers. animals = { :mammals => 'horse’, :carnivore => ‘dinosaur’ :pet => [‘cat’,’dog’] } Getting the value from hashes >> animals[:pet] => [‘cat’,’dog’] You can create empty hash by new class method: months = Hash.new or months = {} You can see all keys names by: keys = months.keys You can see values name by: hash.key
